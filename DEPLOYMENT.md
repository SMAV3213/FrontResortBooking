# 🚀 Инструкция по развертыванию адаптивного фронтэнда\n\n## ✅ Перед развертыванием\n\n### 1. Локальная проверка\n\n```bash\n# Установка зависимостей\ncd frontend\nnpm install\n\n# Запуск dev сервера\nnpm run dev\n\n# В браузере проверить\nhttp://localhost:5173\n\n# Открыть DevTools (F12)\n# Ctrl+Shift+M для эмуляции мобильного\n# Проверить разные размеры\n```\n\n### 2. Проверка на реальном устройстве\n\n```bash\n# Узнать IP компьютера\nipconfig | findstr IPv4\n# Пример: 192.168.1.100\n\n# На смартфоне в браузере\nhttp://192.168.1.100:5173\n\n# Проверить на реальном iPhone/Android\n```\n\n### 3. Проверка по чек-листу\n\nСм. файл `TESTING_CHECKLIST.md`\n\n- [ ] Все размеры проверены\n- [ ] Нет горизонтального скролла\n- [ ] Все кнопки касаемые\n- [ ] Текст читаемый\n- [ ] Модальные окна адаптивны\n\n## 🔨 Сборка для production\n\n### Build\n\n```bash\ncd frontend\n\n# Очистка\nrm -rf dist\n\n# Сборка\nnpm run build\n\n# Проверка\nls -la dist/\n# Должны быть файлы index.html, js/*, css/*\n```\n\n### Preview\n\n```bash\n# Локальный preview production версии\nnpm run preview\n\n# Откройте в браузере\n# http://localhost:4173\n\n# Проверьте на мобильном эмуляторе\n```\n\n## 🐳 Docker (если используется)\n\n### Build Docker image\n\n```bash\ncd frontend\ndocker build -t resort-booking-frontend:latest .\n\n# Проверить\ndocker images | grep resort-booking-frontend\n```\n\n### Run Docker container\n\n```bash\ndocker run -p 80:80 -p 443:443 resort-booking-frontend:latest\n\n# Проверить\ncurl http://localhost\n```\n\n### Docker Compose\n\n```bash\n# В корне проекта\ndocker-compose up -d frontend\n\n# Логи\ndocker-compose logs -f frontend\n\n# Остановить\ndocker-compose down\n```\n\n## 📦 Deployment\n\n### 1. На сервер через SCP\n\n```bash\n# Сборка\nnpm run build\n\n# Копирование на сервер\nscp -r dist/ user@server:/var/www/resort-booking/\n\n# На сервере (nginx)\nsudo systemctl restart nginx\n```\n\n### 2. На Vercel (рекомендуется)\n\n```bash\n# Установка CLI\nnpm install -g vercel\n\n# Логин\nvercel login\n\n# Deploy\ncd frontend\nvercel --prod\n\n# URL будет создан автоматически\n```\n\n### 3. На GitHub Pages\n\n```bash\n# В package.json добавить\n\"homepage\": \"https://username.github.io/resort-booking\"\n\n# Сборка\nnpm run build\n\n# Deploy\ngh-pages -d dist\n```\n\n## 🔒 Security Checklist\n\n- [ ] Нет console.log в production\n- [ ] Нет API ключей в коде\n- [ ] HTTPS включен\n- [ ] CORS настроен правильно\n- [ ] CSP headers установлены\n- [ ] X-Frame-Options установлен\n\n## 📊 Performance Monitoring\n\n### Lighthouse\n\n```bash\n# Chrome DevTools → Lighthouse\n# Цели:\n# Performance: > 90\n# Accessibility: > 95\n# Best Practices: > 90\n# SEO: > 90\n```\n\n### Web Vitals\n\n```bash\n# Проверить:\n# LCP (Largest Contentful Paint): < 2.5s\n# FID (First Input Delay): < 100ms\n# CLS (Cumulative Layout Shift): < 0.1\n```\n\n## 🐛 Troubleshooting\n\n### Проблема: Горизонтальный скролл на мобильном\n\n```bash\n# Проверить\ngrep -r \"width: 110%\" src/\ngrep -r \"position: absolute\" src/\ngrep -r \"right: -\" src/\n\n# Исправить: убрать overflow-x\n```\n\n### Проблема: Зум на iOS input\n\n```scss\n// Проверить что все inputs имеют\ninput, textarea {\n  font-size: 16px;\n}\n```\n\n### Проблема: Медленная загрузка\n\n```bash\n# Оптимизировать изображения\nnpm install -g imagemin-cli\nimagemin src/assets/*.{jpg,png} --out-dir=optimized/\n\n# Проверить bundle size\nnpm run build -- --analyze\n```\n\n## 🔄 Continuous Integration\n\n### GitHub Actions\n\nСоздать `.github/workflows/deploy.yml`:\n\n```yaml\nname: Deploy Frontend\n\non:\n  push:\n    branches: [main]\n  pull_request:\n    branches: [main]\n\njobs:\n  build-and-deploy:\n    runs-on: ubuntu-latest\n    \n    steps:\n      - uses: actions/checkout@v2\n      \n      - name: Setup Node.js\n        uses: actions/setup-node@v2\n        with:\n          node-version: '18'\n      \n      - name: Install dependencies\n        run: npm ci\n        working-directory: ./frontend\n      \n      - name: Build\n        run: npm run build\n        working-directory: ./frontend\n      \n      - name: Deploy\n        run: |\n          # Развернуть на сервер\n          echo \"Deploying...\"\n```\n\n## ✨ Post-Deployment Checklist\n\n### Проверка на live сервере\n\n- [ ] Сайт доступен (HTTP 200)\n- [ ] Все изображения загружаются\n- [ ] CSS/JS загружаются без ошибок\n- [ ] Нет 404 ошибок в консоли\n- [ ] API endpoints доступны\n- [ ] HTTPS работает\n- [ ] Редирект с HTTP на HTTPS\n\n### Мобильная проверка (live)\n\n- [ ] На iPhone SE (375px)\n- [ ] На Android телефоне\n- [ ] На планшете\n- [ ] В браузере Chrome\n- [ ] В браузере Safari (iOS)\n- [ ] На медленном 3G (DevTools Throttle)\n\n### Функциональность\n\n- [ ] Регистрация работает\n- [ ] Логин работает\n- [ ] Поиск номеров работает\n- [ ] Бронирование работает\n- [ ] Личный кабинет доступен\n- [ ] Профиль редактируется\n- [ ] Админ панель работает (если применимо)\n\n## 📝 Логирование\n\n### Мониторить ошибки\n\n```javascript\n// Добавить в App.tsx\nimport * as Sentry from \"@sentry/react\";\n\nSentry.init({\n  dsn: process.env.REACT_APP_SENTRY_DSN,\n  environment: process.env.NODE_ENV,\n});\n```\n\n### Проверить логи\n\n```bash\n# На сервере\ntail -f /var/log/nginx/error.log\ntail -f /var/log/nginx/access.log\n\n# В браузере\n# F12 → Console → Filter: Errors\n```\n\n## 🔄 Откат\n\nЕсли что-то пошло не так:\n\n```bash\n# Откат на предыдущий коммит\ngit revert HEAD\n\n# Или переключиться на стабильный тег\ngit checkout v1.0.0\n\n# Пересобрать\nnpm run build\n\n# Разместить снова\n```\n\n## 📞 Техподдержка\n\nПри проблемах на production:\n\n1. **Проверить логи**\n   ```bash\n   docker logs resort-booking-frontend\n   ```\n\n2. **Перезагрузить контейнер**\n   ```bash\n   docker restart resort-booking-frontend\n   ```\n\n3. **Очистить кэш браузера**\n   -
